Cascaded Style Sheets (CSS)

Internal CSS

We could add css styles internally to a page element using the tag style. We apply styles to an element in the head section so that the style will be applied to the element automatically when used in that page.

External CSS

We should place the styling of elements in a separate stylesheet and link them to the pages so that the whole website could be styles consistently. If we want to style any pages or element we could do that too. The styles specfied at element will have higher precedence over the page level or global level.

  • Tag Selectors -> Styles the elemets all over
  • Class Selectors -> Define classes with styles and apply the classes to elements. Class selectors can be cascaded. Defined using "." operator. A page can have any number of elemets with the same class. Syntax: .classname { properties }
  • ID selectors -> Define ID with styles and apply to the element with that ID. A page can have only one element with the ID. ID cannot be cascaded. Defined using "#" operator. Syntax: #IDname { properties }

CSS display Property

  • display: block
    • take up the entire width of the screen
    • can specify custom widths
    • Examples: h1-h6, p, div
  • display: inline
    • Placed inline to other elements
    • Examples: Span, Img, Anchor
    • Cannot specify widths
  • display: inline-block
    • Can specify widths, wont fill the entire screen - can sit in same line with other elemts
  • display: none
    • The element will be GONE from the view
    • Similar to Android : visibility:GONE
    • For getting android: visibility: invisible , use property "visibility:hidden"

CSS positioing

  • static
    • Default positioning of element within the container
  • relative
    • by changing the position property to relative, we can modify the cordinates (left,top,right,bottom)
    • the element poition is changed w.r.t to its default position - NOT w.r.t edge of screen!
    • the element does not pushes the neighbouring elements (like margin does)
    • When Using relative positioning, eventhough the element's position changes, its like the old or default position is left as a placeholder so that the other elements below or above it are not affected - as if that element's ghost is still there
  • absolute
    • achieved by changing position property to absolute
    • the position changes w.r.t to the parent container
    • the element shifts completely - no ghosting..hence the other elements around re adjusts their positioning
    • The parent is by default the body. To make a container as parent, that container need to mark its position as relative
  • fixed
    • By using this position, we can fix an element w.r.t to the container
    • even when other elements in the container scrolls, the fixed element will stay fixed.
